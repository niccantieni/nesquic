# Requirements: Quinn IUT

This document describes the behavior and architecture of the Quinn implementation under test (IUT) in Nesquic. It serves as the reference specification for adding new QUIC library integrations.

---

## 1. Project Overview

Nesquic is a testbed for comparing QUIC library implementations under realistic network conditions. Each library is wrapped in an **IUT** (Implementation Under Test) that exposes a standardized interface, allowing the harness to drive all implementations identically.

The workspace is structured as:

```
nesquic/
  iut/
    quinn/        # Quinn IUT (this document)
    quiche/       # Quiche IUT
    neqo/         # Neqo IUT (WIP)
  utils/          # Shared traits, protocol, and tooling
  nesquic/        # Test harness
  res/pem/        # TLS certificates (generated by rcgen tool)
```

The Quinn IUT (`iut/quinn/`) is the reference implementation. Every other IUT must satisfy the same traits and protocol contract.

## 2. Shared Library Layer (`utils/`)

The `utils` crate defines the interfaces and protocol that all IUTs must implement. It has no knowledge of any specific QUIC library.

### 2.1 Core Traits — `utils/src/bin.rs`

All IUTs implement two traits: `bin::Client` and `bin::Server`.

#### `ClientArgs`

Parsed from the CLI via `clap`. Fields:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `url` | `Url` | `https://127.0.0.1:4433` | Server address to connect to |
| `unencrypted` | `bool` | `false` | Perform TLS handshake but skip encryption |
| `cert` | `String` | — | Path to the TLS certificate PEM file (used as root CA) |
| `blob` | `String` | — | Desired response size, e.g. `"50Mbit"` or `"2Gbit"` |

#### `ServerArgs`

Parsed from the CLI via `clap`. Fields:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `unencrypted` | `bool` | `false` | Perform TLS handshake but skip encryption |
| `key` | `String` | — | Path to the TLS private key PEM file |
| `cert` | `String` | — | Path to the TLS certificate chain PEM file |
| `listen` | `SocketAddr` | `0.0.0.0:4433` | Address and port to listen on |

`key` and `cert` are mutually required (clap `requires`).

#### `bin::Client` trait

```
fn new(args: ClientArgs) -> Result<Self>
async fn connect(&mut self) -> Result<()>
async fn run(&mut self) -> Result<()>
```

An IUT client must be constructible from `ClientArgs`, must support an explicit connect step, and must execute one request/response cycle in `run()`.

#### `bin::Server` trait

```
fn new(args: ServerArgs) -> Result<Self>
async fn listen(&mut self) -> impl Future<Output = Result<()>>
```

An IUT server must be constructible from `ServerArgs` and must block in `listen()` serving incoming connections indefinitely.

### 2.2 Wire Protocol — `utils/src/perf.rs`

The performance test uses a minimal binary protocol over a single bidirectional QUIC stream.

#### `Request`

Represents a client request. The client specifies how many bytes it wants the server to send back.

- **Serialization**: 8 bytes, big-endian `usize` encoding of the size in bytes.
- **Parsing from string**: Accepts human-readable strings such as `"100Mbit"`, `"2Gbit"`, `"512Kbit"`, `"800bit"`. The unit is always in bits; the resulting byte count is `bits / 8`. Supported SI prefixes: `K` (1,000), `M` (1,000,000), `G` (1,000,000,000).

#### `Blob`

Represents the server response payload.

- **Construction**: Built from an 8-byte big-endian header (the same encoding as `Request::to_bytes()`). Also constructible from a `&[u8]` slice (reads first 8 bytes).
- **Iteration**: Implements `Iterator<Item = u8>`, yielding `size` zero bytes. This allows lazy, allocation-free response generation.
- Fields: `size: usize` (total bytes to send), `cursor: usize` (bytes sent so far).

#### `Stats`

A measurement accumulator for repeated transfers.

- `start_measurement()` — records `Instant::now()` and prepares a byte counter for this run.
- `add_bytes(n)` — accumulates `n` bytes into the current measurement's counter.
- `stop_measurement()` — computes elapsed duration and throughput in MB/s for the current run; returns `(Duration, f64)`.
- `durations()` / `throughputs()` — return `MeanWithError` (mean and standard error) across all runs.
- `summary()` — formats a human-readable statistics string.

Throughput is computed as `bytes / 1_000_000 / seconds` (megabytes per second, not megabits).

### 2.3 Low-Level I/O Types — `utils/src/io.rs`

These types are defined for future alternative implementations that drive the QUIC state machine directly (without a high-level async API). They are **not used by the Quinn IUT**.

- `Transmit<'a>` — a pending outgoing datagram (`to: SocketAddr`, `data: &[u8]`).
- `DatagramEvent<'a, ID>` — events produced when processing an incoming datagram: `NewConnection(ID)`, `Known(ID)`, or `Respond(Transmit)`.
- `Connection` trait — synchronous, poll-based stream I/O: `write()`, `read()`, `close()`, `poll()`, `readable_stream()`, `writable_stream()`, `time_to_timeout()`, `on_timeout()`, `is_cleaned()`.

---

### 2.4 Certificate Generator — `utils/src/bin/rcgen.rs`

A standalone binary (`rcgen`) that generates a self-signed TLS certificate for local testing.

- Generates a certificate valid for `127.0.0.1` and `10.0.0.1`.
- Writes the private key to `res/pem/key.pem`.
- Writes the certificate to `res/pem/cert.pem`.
- Both files are in PEM format.

These paths are the defaults used by `ServerArgs::test()` and `ClientArgs::test()`.

## 3. Quinn Library Code (`iut/quinn/src/lib.rs`)

The root of the Quinn IUT. It:

- Declares the `client` and `server` submodules.
- Re-exports `Client` and `Server` as the public API of the crate.
- Provides the shared helper `bind_socket`.

### `bind_socket(addr: SocketAddr) -> Result<std::net::UdpSocket>`

Creates and binds a UDP socket to the given address using the `socket2` crate (for low-level control not available via `std`).

- Creates a `DGRAM`/`UDP` socket in the correct address family (`Domain::for_address`).
- For IPv6 addresses: calls `set_only_v6(false)` to enable dual-stack mode (accepting both IPv4 and IPv6 connections).
- Binds the socket to the given address.
- Converts the `socket2::Socket` into a `std::net::UdpSocket` and returns it.

This function is called by both client and server to create their underlying sockets before handing them to Quinn.

## 4. Client (`iut/quinn/src/client.rs`)

Implements the `bin::Client` trait using the `quinn` crate.

### Struct

```
Client {
    args: ClientArgs,
    conn: Option<quinn::Connection>,
    config: quinn::ClientConfig,
}
```

`conn` is `None` until `connect()` succeeds.

### `new(args: ClientArgs) -> Result<Self>`

1. Loads the root certificate from the PEM file at `args.cert` using `rustls::pki_types::CertificateDer::from_pem_file`.
2. Builds a `rustls::ClientConfig` with that certificate as the only trusted root, with no client authentication.
3. Sets the ALPN protocol to `[b"perf"]`.
4. Wraps the rustls config in `quinn::crypto::rustls::QuicClientConfig` and then in `quinn::ClientConfig`.
5. Stores `args` and `config`; `conn` is `None`.

### `connect(&mut self) -> Result<()>`

1. Resolves the server address by calling `.to_socket_addrs()` on `(host, port)` extracted from `args.url`. Uses the first resolved address. Default port: `4433`.
2. Binds a local socket at `[::]:0` (any IPv6 address, OS-assigned port) using `bind_socket`.
3. Creates a `quinn::Endpoint` from the socket using `TokioRuntime`.
4. Sets the client config as the endpoint's default.
5. Calls `endpoint.connect(remote, host)` and `.await`s the handshake.
6. Stores the resulting `quinn::Connection` in `self.conn`.

### `run(&mut self) -> Result<()>`

Requires `connect()` to have been called (bails if `conn` is `None`).

1. Opens a bidirectional stream with `conn.open_bi().await`.
2. Parses `args.blob` into a `perf::Request` (e.g., `"50Mbit"` → 6,250,000 bytes).
3. Serializes the request to 8 bytes and writes it to the send stream with `write_all`.
4. Calls `send.finish()` to close the send side of the stream, signaling end of request.
5. Reads the entire response with `recv.read_to_end(usize::MAX)`.
6. Validates that the received byte count equals `request.size`. Returns an error if they differ.

## 5. Server (`iut/quinn/src/server.rs`)

Implements the `bin::Server` trait using the `quinn` crate.

### Struct

```
Server {
    args: ServerArgs,
    config: quinn::ServerConfig,
}
```

### `new(args: ServerArgs) -> Result<Self>`

1. Loads the full certificate chain from the PEM file at `args.cert` using `CertificateDer::pem_file_iter` (supports multi-cert chains).
2. Loads the private key from the PEM file at `args.key` using `PrivateKeyDer::from_pem_file`.
3. Builds a `rustls::ServerConfig` with no client authentication and the loaded certificate chain + key.
4. Sets the ALPN protocol to `[b"perf"]`.
5. Wraps the rustls config in `quinn::crypto::rustls::QuicServerConfig` and then in `quinn::ServerConfig`.

### `listen(&mut self) -> Result<()>`

1. Binds a socket to `args.listen` using `bind_socket`.
2. Creates a `quinn::Endpoint` with the server config and the socket using `TokioRuntime`.
3. Logs the local address.
4. Loops: calls `endpoint.accept().await` to get the next incoming `quinn::Incoming`.
5. For each incoming connection, spawns a Tokio task running `handle_connection`.
6. Errors in spawned tasks are logged but do not stop the server loop.

### `handle_connection(conn: quinn::Incoming) -> Result<()>`

A free function (not part of the trait). Runs in a spawned task per connection.

1. Awaits the QUIC handshake: `conn.await`.
2. Loops: calls `connection.accept_bi().await` to get the next bidirectional stream.
3. On `ConnectionError::ApplicationClosed`: exits the loop cleanly (normal client disconnect).
4. On other errors: propagates the error.
5. For each stream, spawns a Tokio task running `handle_request`.
6. Errors in stream tasks are logged but do not stop the connection loop.

### `handle_request((send, recv): (quinn::SendStream, quinn::RecvStream)) -> Result<()>`

A free function. Runs in a spawned task per stream.

1. Reads up to 64 KiB from the recv stream with `recv.read_to_end(64 * 1024)`.
2. Parses the first 8 bytes as a `perf::Blob` (big-endian size).
3. Logs the requested size.
4. Sends the response: iterates the `Blob` (which yields `blob.size` zero bytes) and writes it as a single chunk with `send.write_chunk(Bytes::from_iter(blob))`.
5. Calls `send.finish()` to close the send side of the stream gracefully.

## 6. End-to-End Protocol Flow

```
Client                                    Server
  |                                         |
  |-- QUIC handshake (TLS, ALPN "perf") --> |
  |                                         |
  |-- open bidirectional stream ----------> |
  |-- 8-byte request (size, big-endian) --> |
  |-- FIN (send side closed) ------------> |
  |                                         |-- parse Blob from 8-byte header
  |                                         |-- write blob.size zero bytes
  |                                         |-- FIN (send side closed)
  |<-- response bytes (all zeros) --------- |
  |                                         |
  |-- validate len(response) == size        |
```

Key properties:
- The request is exactly 8 bytes (big-endian `usize` encoding of the desired response size in bytes).
- The response is exactly `size` zero bytes.
- Both sides use a single bidirectional stream per request.
- Multiple requests can be served concurrently: each stream is handled in its own Tokio task.
- Multiple connections can be served concurrently: each connection is handled in its own Tokio task.

## 7. TLS Configuration

| Parameter | Client | Server |
|-----------|--------|--------|
| Library | rustls | rustls |
| Crypto backend | ring | ring |
| ALPN | `"perf"` | `"perf"` |
| Client auth | None | Not required |
| Certificate loading | Single root cert from PEM | Full chain from PEM |
| Key loading | N/A | Private key from PEM |
| Validation | Validates server cert against root | N/A |

Certificates for local testing are generated by the `rcgen` binary and stored in `res/pem/`.
